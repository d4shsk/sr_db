### Таблица «Типы данных и объекты СУБД MySQL»

| № | Тип данных / объекты СУБД | Описание / характеристики |
|---|---------------------------|---------------------------|
| **I** | **ТИПЫ ДАННЫХ (категории информации в системе БД)** | |
| 1 | **Постоянные данные** | Основной набор данных, хранящихся в базе, отражающий состояние объектов и их отношений в предметной области. |
| 2 | **Транзитные данные** | Промежуточные результаты вычислений или работы СУБД, которые не являются частью постоянных данных БД. |
| 3 | **Входные данные** | Информация, передаваемая системе извне (например, пользователем с клавиатуры). Может стать причиной изменения постоянных данных или войти в них. |
| 4 | **Выходные данные** | Сообщения и результаты вычислений, выдаваемые системой пользователю на основе постоянных данных. |
| 5 | **Метаинформация (метаданные)** | «Информация об информации». Данные, описывающие структуру и элементы самой базы данных, пользователей, права доступа и т. д. |
| **II** | **ОБЪЕКТЫ И СТРУКТУРЫ СУБД** | |
| 6 | **База данных (БД)** | Именованная совокупность интегрированных и общих данных. Рассматривается как электронная картотека или хранилище. |
| 7 | **Таблица (реляционная)** | Основной объект хранения данных в реляционных СУБД (таких как MySQL). Данные в ней организованы в виде строк (записей) и столбцов (полей/атрибутов). |
| 8 | **Представление (View)** | Виртуальная структура/объект базы данных, предоставляющая данные в определенном виде (упоминается как объект, описываемый в системном каталоге). |
| 9 | **Системный каталог (Словарь данных / Репозиторий)** | Централизованное хранилище метаинформации, набор служебных таблиц с информацией о структуре баз данных, таблиц, представлений. Обеспечивает свойство «самодокументируемости» БД. |
| 10| **Хранимое поле** | Наименьшая единица хранимых данных в архитектуре БД. База данных содержит экземпляры хранимых полей разных типов. |
| 11| **Хранимая запись** | Логический набор связанных хранимых полей. |
| 12| **Хранимый файл** | Набор всех экземпляров хранимых записей одного типа (структура физического/внутреннего уровня). |
| 13| **Внешнее представление** | Содержимое базы данных, каким его видит отдельный конечный пользователь или группа пользователей. Состоит из внешних записей. |
| 14| **Концептуальное представление** | Представление всей информации базы данных в абстрактной форме, объединяющее данные всех пользователей. Состоит из концептуальных записей. |
| 15| **Внутреннее представление** | Описывает все подробности, связанные с хранением данных в базе. Состоит из внутренних записей. |

## Задание 1.4: Развертывание БД PostgreSQL с помощью Docker

Для развертывания СУБД PostgreSQL в изолированном контейнере Docker использовалась командная строка (CLI). 

### 1. Команда для развертывания и запуск контейнера
Команда скачивает официальный образ PostgreSQL (если его нет локально) и запускает контейнер в фоновом режиме, задавая пароль суперпользователя и пробрасывая стандартный порт:

```bash
docker run --name pg-server -e POSTGRES_PASSWORD=mysecretpassword -p 5432:5432 -d postgres:latest
```

**Результат выполнения команды:**
[![image.png](https://i.postimg.cc/br8XdYXz/image.png)](https://postimg.cc/9RxgnhFS)


### 2. Подключение к СУБД из командной строки
Для проверки работоспособности мы подключаемся к запущенному контейнеру с помощью встроенной утилиты `psql`:

```bash
docker exec -it pg-server psql -U postgres
```

**Результат:**
```text
psql (16.2 (Debian 16.2-1.pgdg120+2))
Type "help" for help.

postgres=# 
```

### 3. Простейшая работа с БД (создание и получение данных)
Выполняем SQL-запросы для создания базы данных, таблицы и добавления записей, чтобы продемонстрировать успешное развертывание:

```sql
-- Создание новой базы данных
CREATE DATABASE sr_test;

-- Подключение к созданной базе
\c sr_test

-- Создание таблицы для хранения данных студентов
CREATE TABLE students (
    id SERIAL PRIMARY KEY,
    name VARCHAR(50)
);

-- Добавление тестовой записи
INSERT INTO students (name) VALUES ('Даниил Шарманов');

-- Получение данных из таблицы
SELECT * FROM students;
```
