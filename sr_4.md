## Задание 4.1: Проектирование с учетом 3НФ

Для приведения исходной электронной таблицы к третьей нормальной форме (3НФ), необходимо устранить частичные и транзитивные зависимости. 

**Анализ исходных данных:**
* Исходный составной первичный ключ в единой таблице — `(OrderNum, CatalogNum)`.
* **Нарушение 2НФ:** Атрибуты `Product` и `Price` зависят только от `CatalogNum` (части ключа). Атрибуты клиента (`CustomerNum`, `CustomerName` и др.) зависят только от `OrderNum`.
* **Нарушение 3НФ:** В данных заказа присутствует транзитивная зависимость: `OrderNum` определяет `CustomerNum`, а `CustomerNum` уже определяет `CustomerName`, `CustomerAddress` и `CustomerPhone`.

Разбив исходную таблицу для устранения этих аномалий, мы получаем 4 взаимосвязанные таблицы в 3НФ:

### 1. Таблица «Товар» (Products)
Хранит независимую информацию о товарах из каталога.
* **CatalogNum** — Каталожный номер товара. **Первичный ключ (PK)**. Обязательно (**NOT NULL**).
* **Product** — Название товара. Обязательно (**NOT NULL**).
* **Price** — Цена за единицу. Обязательно (**NOT NULL**).

### 2. Таблица «Заказчик» (Customers)
Хранит независимую информацию о клиентах, делающих заказы.
* **CustomerNum** — Номер заказчика. **Первичный ключ (PK)**. Обязательно (**NOT NULL**).
* **CustomerName** — Фамилия и инициалы. Обязательно (**NOT NULL**).
* **CustomerAddress** — Адрес заказчика. Обязательно (**NOT NULL**).
* **CustomerPhone** — Телефон заказчика. Обязательно (**NOT NULL**).

### 3. Таблица «Заказ» (Orders)
Хранит общую информацию о факте заказа и привязывает его к клиенту.
* **OrderNum** — Номер заказа. **Первичный ключ (PK)**. Обязательно (**NOT NULL**).
* **CustomerNum** — Ссылка на заказчика. **Внешний ключ (FK)** к таблице `Заказчик`. Обязательно (**NOT NULL**).
* *Связь:* «Заказчик → Заказ» (связь один-ко-многим). Один клиент может сделать множество заказов, но каждый заказ принадлежит строго одному клиенту.

### 4. Таблица «Позиции заказа» (Order_Items)
Разрешает связь «многие-ко-многим» между заказами и товарами, храня количество конкретного товара в конкретном заказе.
* **OrderNum** — Номер заказа. **Часть первичного ключа (PK)**. **Внешний ключ (FK)** к таблице `Заказ`. Обязательно (**NOT NULL**).
* **CatalogNum** — Каталожный номер товара. **Часть первичного ключа (PK)**. **Внешний ключ (FK)** к таблице `Товар`. Обязательно (**NOT NULL**).
* **Quantity** — Количество единиц товара в данном заказе. Обязательно (**NOT NULL**).
* *Связь:* «Заказ → Позиции заказа» (один-ко-многим) и «Товар → Позиции заказа» (один-ко-многим).

---
**Итоги по требованиям задания:**
1. **Первичные ключи (PK) указаны** для каждой таблицы.
2. Все перечисленные атрибуты помечены как обязательные **(NOT NULL)**, так как согласно логике исходной таблицы отсутствие этих данных сделает запись неполноценной.
3. **Установлены внешние ключи (FK)**, поддерживающие ссылочную целостность между сущностями.
4. **Определены связи:** все логические связи сведены к типу «один-ко-многим» через нормализованную структуру. База данных полностью соответствует **3НФ**.

## Задание 4.2: Простые запросы на SQL

Ниже представлены SQL-запросы, написанные на основе вариативного задания 4.1. *(Синтаксис учитывает особенности MS Access, описанные в методичке, например использование оператора `TOP` и масок `*`)*.

**Запрос 1_Авторы без повторений**
```sql
SELECT DISTINCT Author 
FROM Book;
```

**Запрос 2_Авторы без повторений сортировка 1**
(Выводит список авторов без повторений в алфавитном порядке)
```sql
SELECT DISTINCT Author 
FROM Book 
ORDER BY Author ASC;
```

**Запрос 3_Авторы без повторений сортировка 2**
(Выводит без повторения перечень авторов и годов издания. Сортировка по автору по возрастанию, затем по году по убыванию)
```sql
SELECT DISTINCT Author, BookYear 
FROM Book 
ORDER BY Author ASC, BookYear DESC;
```

**Запрос 4_Книги сортировка**
(Информация о книгах от самых старых к новым, при совпадении года — по названию в алфавитном порядке)
```sql
SELECT * 
FROM Book 
ORDER BY BookYear ASC, Title ASC;
```

**Запрос 5_Две последние книги**
(Вывод двух самых новых изданий)
```sql
SELECT TOP 2 * 
FROM Book 
ORDER BY BookYear DESC;
```

**Запрос 6_Половина списка**
(Вывод первой половины списка книг, упорядоченного по возрастанию года)
```sql
SELECT TOP 50 PERCENT * 
FROM Book 
ORDER BY BookYear ASC;
```

**Запрос 7_Запрос к связанным таблицам Inner**
(Внутреннее соединение таблиц Book и BookInLib)
```sql
SELECT BookInLib.LibID, Book.Author, Book.Title 
FROM Book 
INNER JOIN BookInLib ON Book.BookID = BookInLib.BookID;
```

**Запрос 8_Запрос к связанным таблицам Left**
(Левое внешнее соединение — выводятся все книги, даже те, которых нет в таблице BookInLib)
```sql
SELECT BookInLib.LibID, Book.Author, Book.Title 
FROM Book 
LEFT JOIN BookInLib ON Book.BookID = BookInLib.BookID;
```

**Запрос 9_книги с 1997 по 2001**
```sql
SELECT Author, Title, Publisher, BookYear 
FROM Book 
WHERE BookYear BETWEEN 1997 AND 2001;
```

**Запрос 10_книги с 1997 по 2001 IN**
(Фильтрация по году и списку издательств с помощью IN)
```sql
SELECT Author, Title, Publisher, BookYear 
FROM Book 
WHERE BookYear BETWEEN 1997 AND 2001 
  AND Publisher IN ('Азбука', 'Политехника');
```

**Запрос 11_книги с 1997 по 2001 OR**
(Эквивалент предыдущего запроса с использованием OR)
```sql
SELECT Author, Title, Publisher, BookYear 
FROM Book 
WHERE BookYear BETWEEN 1997 AND 2001 
  AND (Publisher = 'Азбука' OR Publisher = 'Политехника');
```

**Запрос 12_Авторы Like**
(Выводит в обратном алфавитном порядке названия книг, выпущенных после 1999 года, автор которых начинается на «Г» или издательство заканчивается на «а»)
```sql
SELECT Title 
FROM Book 
WHERE BookYear > 1999 
  AND (Author LIKE 'Г*' OR Publisher LIKE '*а')
ORDER BY Title DESC;
```
*(Примечание: в стандарте SQL маской является `%`, но в задании используется синтаксис Access с `*`)*.

### Анализ запроса из пункта 4 (стр. 4)
**Код запроса:**
```sql
SELECT * 
FROM Book 
WHERE Author LIKE 'А*' OR BookYear > 2000 AND NOT Publisher LIKE '[И,П]*';
```
**Описание работы запроса:**
Данный запрос выводит всю информацию о книгах, для которых выполняется **хотя бы одно из двух** условий (так как приоритет оператора `AND` выше, чем у `OR`):
1. Фамилия автора начинается на букву «А» (`Author LIKE 'А*'`).
2. **ИЛИ** одновременно выполняются два условия: год издания строго больше 2000 (`BookYear > 2000`) **И** название издательства **НЕ** начинается на буквы «И» или «П» (`NOT Publisher LIKE '[И,П]*'`).
